## Changes

**Managers plumbing** — progression data the Core already computed is no longer discarded:

- `KillLootRecord.Coins`: per-kill coin amounts (reuses the existing roll — no RNG-order change).
- `CombatReward` level-transition fields (character + combat skill) captured in `ApplyReward`; `ActiveGatheringTickResult` gained skill level-transition fields.
- New normalized `GameManager` events `XpAwarded` and `LevelUp` (one event per level crossed), fired from the combat and gathering tick paths; offline rewards deliberately excluded (OfflineReportPanel already covers them).
- `LootPickedUpEvent.Vacuum` flag distinguishes vacuum pickups from manual ones.

**View juice**:

- `CombatFeedbackView`: gold "+N coins" popup per killed mob; critical hits now orange, larger, with a scale-punch pop-in (all tunables serialized).
- `LootBagView`: spawn bounce (scale 0→overshoot→1) and a fly-to-player vacuum tween; circle colliders are counter-scaled so the click area stays constant; `CombatView` routes vacuum pickups through the tween only when no stacks remain (partial pickups keep the bag).
- `GatheringNodeView` + `GatheringView`: sprite-child hit-shake on every swing, crumble-puff on each successful gather, capped per tick against catch-up batches.

**UI**:

- New `LevelUpBannerPanel`: queued gold "LEVEL X!" / "MINING LEVEL X!" banners, always-active root with CanvasGroup invisibility, built by `UIBuilder` and baked into `GameUI.prefab`.
- `MainHudPanel`: XP-bar pulse on every XP award (character and skill XP).

**Tests**: 7 new EditMode tests (ActiveCombatCoordinatorTests ×3, LifeSkillsTests ×2, LootDropManagerTests ×2); full suite green in Unity Test Runner. No Core/Data changes, no save-shape changes, no new deterministic RNG draws.
