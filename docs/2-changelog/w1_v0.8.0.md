## Changes

- **Ground loot**: killed mobs now drop a physical loot bag on their tile (per defeated actor, AoE-safe). Click the bag to walk over and pick it up; the AutoLoot toggle now means "auto-vacuum after a delay" instead of "route to bank". Unclaimed bags despawn on a tunable timer (blink warning) and are cleared on map travel/quit — ground loot is runtime-only, no save-format change.
- **Inventory-first everywhere**: pickups, gathering, and offline rewards all land in the character inventory; coins stay bank-bound. Overflow stays in the bag until despawn (active) or is reported (offline report now says INVENTORY FULL). Gathering overflow warns in the feed ("lost").
- **Loot lifecycle service**: new `LootDropManager` (Managers) owns ground-loot records, partial pickups, vacuum scheduling (throttled retries), and expiry; `GameManager` owns the commit path (`SpawnLoot`/`TryPickupLoot`, new `LootPickedUp` session event) and relays lifecycle events to View/UI.
- **Feedback layer**: HUD loot feed (pickups, crafted items, gathering gains, inventory-full/expiry warnings) + subtle world-space popups for picked-up loot, gathering gains, and craft outputs; small red "Miss" popups for failed gathering swings and missed attacks (player and enemy). New `CraftCompleted` event.
- **HUD**: dedicated AutoLoot toggle button ("Loot ON/OFF", distinct from Auto-combat); HUD width reflowed for ten buttons.
- **Content**: slime drop table reworked to `slime_goo` (+ occasional small HP potion), authored as ScriptableObject overrides (`Monster_Slime` + `DropTable_Slime` in the ContentRegistry) — closes the kill → loot → craft Leather-armor loop on FirstMap. `wolf_pelt` removed from slime.
- **Drop-table format**: tertiary drops (independent fixed-chance rolls per kill) supported end-to-end — types, stochastic + expected-value rolls (offline parity), ScriptableObject authoring, content validation. Structure only, no tertiary content yet.
- **World view**: new `LootBagView` + `LootBagSortController` (Assembly-CSharp sort bridge mirroring `EnemySortController`); `PlayerController` bag click-targets with arrival/cancel intent signaling; destroyed-bag edge cases resolve safely.
- **Tests**: 17 new/updated EditMode tests (LootDropManager transactions, tertiary rolls/EV/validation, per-actor kill loot, offline + gathering destination regressions); PROJECT.md Flow A updated to the bag-based loot path.
