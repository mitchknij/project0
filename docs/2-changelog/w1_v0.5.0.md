## Changes

Tile patterns become the primary spatial language for new area skills (plan: `docs/1-plans/F_0.5.0_tile-first-targeting.plan.md`).

- **Data contracts**: `SkillTargetingKind` gains `TilePatternAroundSource`/`TilePatternAroundTarget`; new `TilePatternKind` (SingleTile, Cross, SquareRadius, CustomOffsets), `TilePatternFloorPolicy` (SameFloor), and `TilePatternDef` (pattern, size, custom offsets, MaxTargets) on `ClassSkillDef`. New `TileAreaResolved` combat event carrying ordered affected tiles + floor + resolved actor ids — spatial resolution only; `DamageApplied` stays authoritative for outcomes.
- **Core resolver**: new pure `TilePatternResolver` (`Assets/Scripts/Core/Combat/`) — deterministic anchor-first tile generation (Cross rings N/E/S/W, SquareRadius row-major), unique actor resolution filtered by floor/faction/alive/targetable, ordered by tile order then ActorId, capped after ordering.
- **Engine integration**: `ActiveSim` resolves tile skills on the immediate and scheduled paths; scheduled impacts capture the anchor at cast time and — fixed during review — source-anchored pending impacts survive their incidental target's death (target-anchored still cancel). `AutoCombatPolicy` gates tile skills on `MinimumAutoTargets` within the pattern; slot priority unchanged.
- **Offline**: `Activity.cs` slot-order approximation now includes tile skills with a conservative expected-targets estimate = min(pattern tile count, density-derived, MaxTargets), recorded in `AssumptionSource`.
- **Content**: two starter skills — **Shatterstep** (id `ground_smash`; self-anchored Cross 1, Physical, auto-gate 2 enemies; renamed from "Ground Smash" to avoid collision with the existing Ground Slam) and **Arcane Detonation** (target-anchored SquareRadius 1 = 3×3, Arcane). `ContentValidator` validates pattern authoring (kind/pattern pairing, sizes, duplicate/out-of-range offsets, floor policy).
- **Presentation**: `CombatFeedbackView` renders `TileAreaResolved` as short-lived fading diamond outlines (Inspector-configurable tint/duration/offset/size) via the new `GridPathfinder.TryGetTileWorldPosition` conversion seam; one shared overlay material (leak fixed during review).
- **Tests**: 28 new EditMode tests (resolver ordering/filtering, ActiveSim integration incl. single reward transaction and captured-anchor scheduling, auto-gating, offline estimates, content validation); suite 103/103 green. Play-mode acceptance verified by user (Shatterstep cross + Arcane Detonation 3×3, overlays match hits).
- **Known items**: pre-existing PlayMode failure `CombatSpatialAdapter_CapturesLogicalActorsWithoutSpriteGeometry` (v0.4.0 `CombatTargetView.Awake` identity race, documented in `docs/STATE.md`); coverage debt entry for `ContentValidator` pairing checks (`docs/4-unit-tests/COVERAGE-DEBT.md`).
