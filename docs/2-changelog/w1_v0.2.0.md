# Changelog - Week 1, 16-07-2026, V. 0.2.0

**Release Date**: Week 1, 16-07-2026 at 14:58
**Version**: 0.2.0 (previously 0.1.0)
**Object**: feat: rebuild UI layer on procedural theme with hardened boot and bake pipeline
**Code review**: `docs/3-code-review/CR_w1_v0.2.0.md` (Codex loop, 2 rounds -> APPROVED)

## Changes

- **Procedural-only theme**: all PNG-art loading removed from UI code (`PreferSlicedArt`, HUD sheet, nav icons, `SkillIcon` dead code); `UITheme` now serves only generated 9-slice frames + the two pixel fonts. Art files stay on disk for a future reskin.
- **Centralized layout**: new `UITheme.Layout` constants block replaces scattered magic numbers; HUD width is an exact constant formula (no more `745f`).
- **HUD rebuilt**: 7 text nav buttons (Inventory, Equipment, Activity, Travel, Crafting, Talents, Bank — every panel reachable) + Auto toggle + 3 skill slots; responsive width clamp (`MainHudPanel.ClampWidthToCanvas`) with min-width locks so only the center block shrinks on narrow aspects.
- **Boot ordering hardened**: `UIBuilder.Bootstrap` fallback is deferred behind new `SceneLoader.InitialLoadCompleted`; the baked `GameUI.prefab` is authoritative, runtime build only fires when no `UIBootstrapper` exists (legacy direct-Play scenes keep working).
- **Wiring hardened**: `MainHudPanel.BindButton` uses direct references only (name-string fallback removed); dead `PanelManager.ShowModal/HideModal` deleted with zero-caller evidence.
- **Bake gate fixed**: `UIBakeTool` now rejects persistent sprites/fonts outside `Assets/Art/UI/Generated/` (previously silently skipped), so a successful bake proves no art reference survived. All bake strings English.
- **Known issue (accepted at release)**: the baked `GameUI` instance sits in `Maps/FirstMap.unity` instead of `PersistentGame.unity` (wrong active scene during bake) — UI unloads on map travel until moved. See CR requester note + `docs/STATE.md`.
