## Changes

- **Mining live in FirstMap**: first playable gathering nodes ever placed — copper rocks (`Rock_1.prefab`, `copper_vein`) and one choppable tree variant (`Tree_0.prefab`, `oak_tree`); rock prefabs generated from `Rock.png` via `WorldAssetPrefabGenerator` on the `Tree_Base` profile.
- **Per-swing progress (Core)**: `ActiveGatheringResult` gains `ActionIntervalMs` / `ActionProgress01`; `LifeSkills.Tick` populates them deterministically and no longer accrues swing time while walking to the node (no retroactive swings on arrival).
- **World-space progress bar (View)**: `GatheringNodeView` renders an amber two-sprite bar (CombatTargetView pattern, Inspector-tunable, world-bounds aware for offset Visual children); `GatheringView` tracks the clicked node instance (multiple rocks share one node id), drives/hides the bar, and warns when an activity assignment is silently rejected (map/level).
- **Crafting scope change (user-approved, diverges from store.ts)**: materials draw **bank-first, then character inventory**; coins stay bank-only; crafted output lands in the **character inventory**; `CanCraft` dry-runs the transaction so the Craft button honestly reports "Inventory is full".
- **Scene repair**: restored `ProjectedSpriteShadow` on all 84 FirstMap trees (lost to a stale-Editor save) and cleaned stray components off the `Rocks`/`World Objects`/`Enemies` containers.
- **Tests**: +4 `LifeSkillsTests` (progress/interval, out-of-range), +5 new `CraftingTests` (inventory-fallback crafting, bank-first split, coins gate, purity, full-inventory rejection); `GameplayLoopSmokeTests` crafting assertions updated to the new output destination.
- **Version lineage**: v0.6.0 (skillbar eight slots) remains planned/unreleased; this release skips to keep plan-file numbering stable.
